# 善见 Agent Demo Script

1. 打开 `/projects`，先讲一句定位：这是给有资质公益机构用的救助项目 Agent，不是个人筹款平台。
2. 在首页首屏展示“混乱求助材料 -> 证据链与四辨 -> 脱敏公示 -> 钱物服匹配”的 5 分钟闭环。
3. 在“获奖 Demo 中心”点击“运行本地LLM Agent”，说明默认接入 `http://127.0.0.1:3000/v1/chat/completions`，本地服务不可用时会降级为离线兜底剧本。
4. 展示材料智能处理台：OCR抽取、金额矛盾、隐私脱敏三件事要在 20 秒内讲清楚。
5. 展示 PMF 证据：节省个案材料整理时间、覆盖四个直接受益岗位、同时打到六个评分维度。
6. 指出右上角三个入口：求助申请入口、机构四辨工作台、捐助意向管理。
7. 进入 `/apply` 求助申请入口，展示大病救助材料如何被 AI 结构化，尤其是“受助人真实需要”和“我不会整理材料，先写一段话”。
8. 进入 `/workbench` 机构四辨工作台，运行四辨审核。
9. 解释 AI 只给证据、风险、优先级和复核清单，最终由机构工作人员决定。
10. 回到首页，在项目详情页打开 AI问项目，询问“这个项目目前最需要什么？”并说明 AI 只基于脱敏事实回答。
11. 点击我要帮助或进入 `/intentions` 捐助意向管理，说明平台只登记钱/物/服三类帮助意向，不在平台内收款。
12. 展示 AI 如何把帮助意向匹配到治疗费用、政策咨询、交通陪诊等真实需要，并生成机构跟进话术。
13. 结束时讲商业落地和加分项：机构年费 + 项目包 + 私有化部署，准备 ClawHunt 上架材料和游园展示话术。
